# Portfolio Site Redesign: Animations, Dark Mode, SEO & Navigation

## Problem

The portfolio site at arush361.github.io was a static Jekyll site with plain text, no animations, broken cross-page navigation, zero SEO optimization, and no dark mode. The site didn't reflect the engineering depth of the portfolio owner.

## Solution

Complete redesign across 10 areas, all in vanilla CSS/JS (no frameworks).

### 1. Interactive Terminal Intro

Replaced prose bio paragraphs with an animated terminal simulation:

```javascript
var script = [
  { type: 'cmd', prompt: '>', text: ' arush.role()' },
  { type: 'output', text: '<span class="t-key">Sr. Product Manager</span>' },
  // ... lines render sequentially with staggered delays
];
```

- Commands have typing delay (400ms), outputs appear faster (120ms)
- Color-coded: green prompts, yellow highlights, blue stats, grey comments
- `<noscript>` fallback with identical content for SEO crawlers
- `.sr-only` hidden text block for search engine indexability

### 2. Background Animations

**Particle Constellation Canvas:**
- Full-viewport `<canvas>` with ~120 interactive particles
- Particles connect with lines when within 180px distance
- Mouse interaction: repulsion + connection lines on hover
- Dark mode switches to cyan/teal particles (`rgba(0, 220, 255)`)
- Hidden on mobile (`< 48rem`) for battery savings
- Pauses via `visibilitychange` when tab is hidden

**Floating Code Glyphs:**
- Domain-relevant tokens (JWT, SSO, RBAC, MCP, API, async) drift upward
- Spawned via `setInterval` every ~2.5s at 12% opacity
- CSS `@keyframes glyphDrift` handles the animation

**Aurora Gradient:**
- Three blurred gradient blobs behind hero section
- CSS-only animation with `transform: translate()` for GPU acceleration

### 3. Dark Mode

```css
:root {
  --bg: #e5e5e5;
  --paper: #fff;
  --text: #1a1a1a;
  /* ... */
}

[data-theme="dark"] {
  --bg: #111;
  --paper: #1a1a1a;
  --text: #e0e0e0;
  /* ... */
}
```

- Toggle button (moon/sun) fixed at bottom-right
- Persists via `localStorage.setItem('theme', next)`
- Respects `prefers-color-scheme: dark` on first visit
- Smooth transition on all theme-aware elements

### 4. Scroll Reveal & Stagger

- IntersectionObserver triggers `.visible` class on scroll
- Staggered children via `transition-delay: calc(n * 0.07s)`
- `fixRevealGaps()` runs on init to instantly show near-viewport elements, preventing blank gaps

### 5. Navigation Fixes

- Anchor links changed from `#experience` to `/#experience` (absolute paths work from any page)
- Animated underline on hover via `::after` pseudo-element
- Active state detection for current page
- Header shrinks on scroll (padding reduction)

### 6. Product Pages Redesign

- Renamed `/projects/` to `/products/` across all files
- Plain text rows replaced with 2x2 card grid with icons, tags, hover animations
- Animated tab switching with fade transition
- Detail pages: hero gradient, list item stagger, screenshot hover zoom, metric card pulse

### 7. SEO Overhaul

- `sitemap.xml` with all pages and priorities
- `robots.txt` pointing to sitemap
- `<link rel="canonical">` on every page
- JSON-LD Person schema on homepage
- Twitter/X card meta tags
- `<meta name="theme-color">`
- OG tags with full absolute URLs

### 8. Config Centralization

Moved all hardcoded values to `_config.yml`:
- `google_analytics`, `formspree_id`, `email`, `medium_url`, `resume_file`
- Templates use `{{ site.variable }}` instead of raw strings
- Dynamic copyright year: `{{ 'now' | date: '%Y' }}`

### 9. Carousel Indicators

- Dot indicators generated by JS based on card count
- Active dot updates on scroll via `track.scrollLeft / cardWidth`
- Click-to-jump functionality

### 10. UX Polish

- Back-to-top button (appears after 400px scroll)
- Classic toggle repositioned on mobile (static instead of fixed overlay)
- Carousel dots for scroll position awareness

## Prevention Strategies

- **Centralize config**: All site metadata in `_config.yml`, never hardcode in templates
- **SEO from day one**: Add sitemap, canonical, structured data before launch, not after
- **Accessibility baseline**: Add `prefers-reduced-motion`, focus states, and screen reader fallbacks during initial build
- **Use `relative_url` filter**: Jekyll's `{{ '/path/' | relative_url }}` prevents broken links after restructuring
- **Test JS-injected content**: Ensure all user-visible text exists in HTML source for crawlers
- **Exclude `_site/` from git**: Use `.gitignore` to prevent stale build artifacts from being committed
- **Cache busting**: Use query params or Jekyll `site.time` for asset versioning during development

## Key Patterns

| Pattern | Used For |
|---------|----------|
| CSS Custom Properties + `[data-theme]` | Dark mode without JS style manipulation |
| IntersectionObserver | Scroll reveal, HR draw, card stagger, counters |
| `requestAnimationFrame` | Particle canvas, counter animations |
| `<noscript>` + `.sr-only` | SEO fallback for JS-injected content |
| `localStorage` | Theme persistence across sessions |
| Jekyll `_config.yml` variables | Centralized configuration |
